# 执行引擎

# 一、基本介绍

## 1、作用

将字节码指令解释/编译为对应平台上的本地机器指令

## 2、组成

### 1) 解释器

根据程序计数器的指令地址，逐条将字节码指令“翻译”为本地机器指令，以便程序能够运行。

### 2) 即时编译器 JIT

### 3) GC



## 3、执行过程： 

 a.首先通过程序计数器找到需要执行的字节码指令 

 b.然后通过虚拟栈的栈帧中的局部变量表中的地址找到堆空间中的对象的实例数据，然后通过实例数据中的对象头中的对象的类型指针找到方法区的类元信息。
